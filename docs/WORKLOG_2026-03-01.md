## 1) Plateforme livree

### 1.1 Hub (`@superhub/hub`)

Fonctionnalites principales:
- Endpoint WS: `/ws`
- API HTTP: `/api/*`
- Registry presence clients/services
- Pub/Sub
- RPC (`rpc_req` / `rpc_res`)
- State store (`state_set`, `state_patch`, `state_get`, `state_watch`)
- Observabilite (logs JSON, metrics, diagnostics, inspector)
- Persistance SQLite optionnelle (presence/state/audit)
- Console web servie via `/console`

### 1.2 SDK TypeScript (`@superhub/sdk`)

- Client unique Node/Web
- Reconnect + backoff + jitter
- API: `publish`, `subscribe`, `rpc`, `onRpc`, `getState`, `setState`, `patchState`, `watchState`
- Support TLS local (CA file ou insecure dev)
- Exemples Node + React embarques dans le package

### 1.3 Contrats (`@superhub/contracts`)

- Envelope unifiee `HubEnvelope`
- Schemas Zod par type de message
- Validation par `name + schemaVersion`
- Normalisation des erreurs

## 4) API de controle ajoutee

Endpoints:
- `GET /api/control`
- `POST /api/control/disconnect`
- `POST /api/control/mute`
- `POST /api/control/unmute`
- `POST /api/control/inbound`
- `POST /api/control/state-broadcast`
- `POST /api/control/clear-queues`

Tous ces endpoints passent par la meme securite que le reste:
- token `X-Hub-Token`
- allowlist subnet

Exemples:

```bash
# Voir etat control plane
curl -sS -H "X-Hub-Token: $HUB_TOKEN" \
  "https://mac-mini-de-olivier.local/api/control"

# Pause inbound global
curl -sS -X POST -H "X-Hub-Token: $HUB_TOKEN" -H "Content-Type: application/json" \
  -d '{"paused":true}' \
  "https://mac-mini-de-olivier.local/api/control/inbound"

# Mute service
curl -sS -X POST -H "X-Hub-Token: $HUB_TOKEN" -H "Content-Type: application/json" \
  -d '{"kind":"service","value":"music"}' \
  "https://mac-mini-de-olivier.local/api/control/mute"

# Unmute topic prefix
curl -sS -X POST -H "X-Hub-Token: $HUB_TOKEN" -H "Content-Type: application/json" \
  -d '{"kind":"topic","value":"iss.*"}' \
  "https://mac-mini-de-olivier.local/api/control/unmute"
```

## 5) Performance & robustesse (haute frequence)

### 5.1 Flux sortant WS

- Mutualisation de serialisation JSON sur fanout multi-recipients
- Gestion plus efficace des sessions queuees

### 5.2 Dedup

- Sweep dedup periodique au lieu de scanner a chaque message

### 5.3 Persistance SQLite

- Snapshots state coalesces en memoire
- Flush periodique batche (au lieu d'un write par update)
- Flush final a la fermeture
- Statements prepares + transaction de flush
- Pragmas SQLite (WAL / synchronous normal)

### 5.4 Rate limiting

- Passage a un token-bucket (plus robuste aux bursts)
- `rateLimitPerMinute`:
  - defaut releve pour flux rapides
  - `0` possible pour desactiver

### 5.5 Diagnostics renforces

- `metrics` expose aussi `bytesIn` / `bytesOut`
- `diagnostics` enrichi:
  - queues
  - routing
  - control state

PIP_CERT="$HOME/.superhub-caddy-root.crt" pip install --extra-index-url "https://mac-mini-de-olivier.local/apps/client/dist/python/simple/" superhub-client

## 8) Correctif ISS updater (Python) - source TLE 403

Symptome rencontre:
- `HTTP Error 403` au fetch Celestrak dans `iss_updater.py`
- process qui s'arrete

Correctif applique:
- plusieurs sources TLE (HTTPS prioritaire)
- retry periodique
- cache local TLE (`~/.superhub/iss_tle.txt`)
- continuation sans crash si source externe indisponible

Env vars supportees:
- `ISS_TLE_URLS` (liste CSV d'URLs)
- `ISS_TLE_CACHE_FILE` (chemin cache local)
- `ISS_SEND_HZ` ou `--hz` (1..50)
